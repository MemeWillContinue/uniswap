# Executor 部署说明

## 已完成

- 合约已编译
- `.env` 已配置（默认测试网）
- 部署成功后会**自动**更新 `script.js` 中的 `EXECUTOR_ADDRESS`

## 你只需一步

1. **填入私钥**：编辑 `uniswap/.env`，在 `DEPLOYER_PRIVATE_KEY=` 后面填上你的 MetaMask 私钥
   - 从 MetaMask：账户详情 → 导出私钥
   - **务必使用测试网钱包**（当前配置为 BSC 测试网）

2. **执行部署**：
   ```bash
   cd uniswap
   npm run deploy
   ```

部署成功后，`script.js` 中的测试网 `EXECUTOR_ADDRESS` 会自动更新，无需手动修改。

---

## 切换到主网

编辑 `.env`：
```
BSC_RPC_URL=https://bsc-dataseed.binance.org
POSITION_MANAGER_ADDRESS=0x46A15B0b27311cedF172AB29E4f4766fbE7F4364
```
然后运行 `npm run deploy`，会更新主网地址到 `script.js`。

---

## 已有合约地址时只更新 script.js

若你已在别处部署好 Executor，只需把地址写入前端配置：

1. 在 `.env` 中设置 `EXECUTOR_ADDRESS=0x你的合约地址`
2. 若为测试网，确保 `BSC_RPC_URL` 为测试网 RPC（含 testnet）；主网则用主网 RPC
3. 执行：`npm run set-executor`

会按当前 `BSC_RPC_URL` 自动更新 `script.js` 中对应网络的 `EXECUTOR_ADDRESS`。
